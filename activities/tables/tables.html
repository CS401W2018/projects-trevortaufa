Engagement

  1. When do you use a table?

    2. When organizing tabular data, meaning data that is best represented in a grid of rows and columns. Examples include:

    • Financial Reports (e.g., sales data, budgets)
    • Schedules (e.g., class timetables, even agendas)
    • Comparison Charts (e.g., product feature comparisons)
    • Data from databases (e.g. employee records, inventory lists)

  2. When do you NOT use a table?

    3. Avoid using:

    • Layout purposes (Use CSS Flex or Grid)
    • Non-tabular content
    • Forms

  3. When do you need to to extra work for table accessibility?

    4. You need to improve accessibility when:

    • Adding headers - Use "th" (table headers)
    • Providing captions - Use "caption" tag to describe the table's content
    • Associating headers and data- Use "id" and "headers" tag attributes for complex tables with multiple layers of headers.
    • Ensuring proper contrast and spacing - Ensure readable text and spacing for better visibility.
