  1. The form tag has an "action" attribute. What does it do?

    The action attribute specifies the URL where the form's data should be sent the when the form is submitted. In this current page, the action is set to "processing.html" meaning that the data is sent to that page. (In this case it is non existent).

  2. The form tag has a "method" attribute. What does it do?

    The method attribute specifies how the form data should be sent to the server when the form is submitted. This is either done through FORM or GET (GET by default.)

  3. What is the difference between a "POST" and a "GET"?

    • POST
      • Sends form data in the request body (not in the URL)
      • Suitable for sensitive data (e.g., passwords) or when sending large amounts of data.
    • GET
      • Appends form data to the URL
      • Suitable for simple searches or when data is not sensitive (e.g. Google searches)

    • When to Use:
      • Use GET for retrieving data (e.g., search forms)
      • Use POST for sumbitting sensitive or large data (e.g., login, registration).
